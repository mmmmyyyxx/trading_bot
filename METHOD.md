# Method

本文档说明本项目的研究方法。项目目标不是直接构建实盘交易系统，而是搭建一个可复现的 A 股量化研究流程：从 AKShare 获取日频行情，转换为 Qlib 数据格式，训练横截面选股模型，并用统一的组合回测与诊断指标评估信号质量。

## 1. 研究对象

本项目以中国 A 股日频数据为基础，当前主要实验使用沪深 300 当前成分股股票池。数据区间在完整基线中覆盖 2018-01-02 至 2024-12-31，并按时间顺序切分为训练、验证和测试阶段：

| Segment | Period | Purpose |
|---|---|---|
| Train | 2018-01-01 to 2021-12-31 | 模型训练 |
| Valid | 2022-01-01 to 2022-12-31 | 参数选择与早停 |
| Test | 2023-01-01 to 2024-12-31 | 样本外回测与评估 |

需要注意，当前沪深 300 股票池来自当前成分股，历史回测仍可能存在成分股幸存者偏差。因此结果应被视为研究基线，而不是严格无偏的历史指数成分回测。

## 2. 数据获取与标准化

数据源由 `scripts/prepare_akshare_data.py` 和 `scripts/run_alpha158_pipeline.py` 调用 AKShare 下载。核心字段包括：

- 价格成交字段：`open`, `high`, `low`, `close`, `volume`, `amount`
- 复权因子：`factor`
- A 股状态字段：`is_st`, `is_paused`, `limit_up`, `limit_down`
- 元数据字段：`list_date`, `industry`

股票代码会统一规范为 `600000.SH`、`000001.SZ` 这类格式，并在写入 Qlib 时转换为 Qlib 使用的内部 symbol。非数值型元数据不会直接写入 Qlib 特征二进制文件，而是保存在 `metadata/instruments.parquet` 或 `metadata/instruments.csv` 中。

## 3. 股票池过滤

项目在 `ashare_adapter.filters` 中实现了向后看的股票池过滤逻辑，避免使用未来信息。每个交易日都会为每只股票生成两个布尔字段：

- `eligible`：基础可交易股票池
- `selected`：最终进入 Qlib 数据加载与信号评估的股票池

基础过滤条件包括：

| Filter | Default | Description |
|---|---:|---|
| ST exclusion | enabled | 剔除 ST 股票 |
| Paused exclusion | enabled | 剔除停牌股票 |
| Minimum listed days | 120 | 剔除上市时间过短的股票 |
| Minimum rolling amount | 10,000,000 | 剔除近期成交额过低的股票 |
| Liquidity window | 20 days | 用过去 20 日成交额均值衡量流动性 |
| Limit-buy exclusion | disabled | 可选剔除开盘涨停、难以买入的股票 |

如果启用 `dynamic_liquidity_top_n`，`selected` 会进一步限制为每日 `eligible` 股票中成交额排名前 N 的股票。当前完整 HS300 基线没有启用动态 top-N，因此 `selected` 等于 `eligible`。Qlib 工作流通过 `ExpressionDFilter` 应用：

```text
$selected > 0.5
```

这意味着模型训练、预测和回测只在过滤后的可交易股票池上进行。

## 4. Qlib 数据转换

`ashare_adapter.qlib_converter` 将清洗后的 A 股行情写入 Qlib 原生 binary 格式。写入的主要数值字段包括：

```text
open, high, low, close, vwap, volume, amount, factor,
is_st, is_paused, limit_up, limit_down,
listed_days, avg_amount, eligible, selected
```

其中 `vwap` 在缺失时由 `amount / volume` 计算，无法计算时回退为 `close`。转换器还会写入：

- `calendars/day.txt`
- `calendars/day_future.txt`
- `instruments/all.txt`
- `features/<symbol>/*.day.bin`
- `metadata/instruments.*`

`day_future.txt` 用于保证 Qlib 在回测边界处可以正确处理未来一个交易步长。

## 5. Alpha158 + LightGBM 基线模型

主基线使用 Qlib 官方风格的 Alpha158 特征处理器和 LightGBM 回归模型。配置文件包括：

- `configs/qlib_alpha158_lgb.yaml`
- 运行时生成的 `reports/<run>/alpha158_lgb_runtime.yaml`

模型形式如下：

```text
features = Alpha158 daily technical factors
label = future return label generated by Qlib
model = LightGBM regression model
objective = mean squared error
```

当前主要超参数包括：

| Parameter | Value |
|---|---:|
| learning_rate | 0.05 |
| max_depth | 8 |
| num_leaves | 210 |
| colsample_bytree | 0.8879 |
| subsample | 0.8789 |
| lambda_l1 | 205.6999 |
| lambda_l2 | 580.9768 |

训练完成后，Qlib 会生成测试期每个交易日、每只股票的预测分数 `score`。该分数代表模型对未来收益的相对排序判断，后续组合构建只使用横截面排序，而不直接解释为绝对收益预测。

## 6. 对照基线

除 Alpha158 + LightGBM 外，项目还包含一个更简单的反转 + 低波动 Ridge 回归基线，配置在：

- `configs/qlib_reversal_lowvol.yaml`
- `configs/qlib_reversal_lowvol_5d.yaml`
- `configs/qlib_reversal_lowvol_20d.yaml`

该基线使用两个手工因子：

```text
short_term_reversal = -1 * ($close / Ref($close, lookback) - 1)
low_volatility = -1 * Std($close / Ref($close, 1) - 1, 60)
```

它的作用是作为低复杂度模型参照，用来判断 Alpha158 + LightGBM 的增益是否来自更丰富的特征和非线性模型，而不仅仅来自市场暴露或回测设定。

## 7. 组合构建与回测

回测使用 Qlib 的 `TopkDropoutStrategy`。策略在每个调仓日按照预测分数排序，持有排名靠前的股票，并逐步替换排名下降的持仓。

主要设置为：

| Setting | Typical Value | Description |
|---|---:|---|
| topk | 20 or 50 | 目标持仓数量 |
| n_drop | 4 or 10 | 每次调仓替换数量 |
| account | 100,000,000 | 初始资金 |
| benchmark | SH000300 | 沪深 300 基准 |
| deal_price | close | 以收盘价成交 |
| open_cost | 0.00031 | 买入成本 |
| close_cost | 0.00081 | 卖出成本 |
| min_cost | 5 | 单笔最低成本 |
| limit_threshold | 0.095 | Qlib 统一涨跌停阈值 |

当前回测仍使用 Qlib 的统一 `limit_threshold`，而不是完整的 A 股分板块涨跌停规则。项目已经将逐股票 `limit_up` 和 `limit_down` 写入数据，后续可以扩展为更精细的订单过滤或自定义 exchange。

## 8. 信号评估

项目同时使用 Qlib 自带记录器和本地诊断脚本评估信号。主要信号指标包括：

- IC：预测分数与未来收益的 Pearson 相关系数
- Rank IC：预测分数与未来收益排名的 Spearman 相关系数
- ICIR / Rank ICIR：IC 均值除以标准差
- 分组收益：按预测分数分为 5 组，比较高分组与低分组收益
- 多周期 forward return：支持 1 日、5 日、20 日收益诊断

相关实现位于：

- `ashare_adapter.diagnostics.compute_ic`
- `ashare_adapter.diagnostics.compute_group_returns`
- `ashare_adapter.result_export.export_alpha158_results`

这些指标用于判断模型是否具有稳定的横截面排序能力。特别是 Rank IC 和分组单调性通常比单次组合收益更能反映选股信号本身的质量。

## 9. 组合评估

组合层面的评估包括绝对收益、相对收益和交易摩擦影响。主要指标包括：

- account total return
- benchmark total return
- excess annualized return with cost
- excess information ratio with cost
- excess max drawdown with cost
- average daily turnover
- total cost
- average positions

项目也支持与多个指数进行比较，包括：

- HS300
- CSI500
- CSI1000

多基准诊断会输出策略收益、基准收益、超额收益、跟踪误差、信息比率、beta、最大回撤和相对回撤。

## 10. 实验产物

一次完整 Alpha158 运行通常会生成以下产物：

```text
mlruns/<experiment_id>/<run_id>/        Qlib / MLflow 原始实验记录
reports/<run>/summary.md               人类可读摘要
reports/<run>/summary.json             结构化摘要
reports/<run>/pred_label_test.csv      测试期预测与标签
reports/<run>/ic_daily.csv             每日 IC
reports/<run>/rank_ic_daily.csv        每日 Rank IC
reports/<run>/factor_group_summary.csv 分组收益摘要
reports/<run>/port_analysis_1day.csv   组合分析
reports/<run>/report_normal_1day.csv   每日账户报告
reports/<run>/universe_diagnostics.csv 股票池过滤诊断
```

`mlruns` 是 MLflow/Qlib 自动生成的实验目录，保存模型、指标、参数和 artifacts；`reports` 是本项目导出的可读分析结果。

## 11. 复现实验流程

运行完整 Alpha158 pipeline 的典型命令为：

```powershell
python scripts/run_alpha158_pipeline.py `
  --symbols-file data/cache/hs300_symbols_full.txt `
  --max-symbols 300 `
  --start-date 2018-01-01 `
  --end-date 2024-12-31 `
  --benchmark-key hs300 `
  --output-dir reports/alpha158_hs300_full
```

如果已有 Qlib 运行记录，可以导出并重新生成诊断：

```powershell
python scripts/export_qlib_records.py `
  --run-dir mlruns/<experiment_id>/<run_id> `
  --bars data/alpha158_hs300_full_bars.parquet `
  --benchmarks data/benchmarks.parquet `
  --output-dir reports/alpha158_hs300_full/qlib_records `
  --apply-mask `
  --run-diagnostics
```

如果需要比较滚动样本外窗口，可运行：

```powershell
python scripts/run_rolling_baselines.py
```

添加 `--execute` 后会实际执行生成的滚动 Qlib 工作流。

## 12. 方法限制

当前方法仍有以下限制：

- 当前 HS300 股票池使用当前成分股回溯，存在成分股幸存者偏差。
- Qlib 回测使用统一涨跌停阈值，尚未完整模拟 A 股不同板块、ST 股票和逐股票涨跌停限制。
- 回测成交价使用 `close`，与真实交易中的撮合、冲击成本和滑点仍有差距。
- 当前结果主要是研究基线，不能直接解释为可实盘交易策略。
- IC 较弱但组合超额表现较强时，需要进一步检查行业暴露、市值暴露、换手、费用和滚动样本稳定性。

后续改进方向包括：使用历史指数成分股、实现更真实的 A 股交易约束、增加行业/风格中性化诊断、扩展滚动窗口检验，以及将信号与组合构建拆分为更细粒度的实验记录。
